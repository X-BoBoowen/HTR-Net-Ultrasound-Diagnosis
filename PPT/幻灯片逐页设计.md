## 第 5 页：语义分割——模型改进

**类型**：技术深度页

**文字内容**：

大标题（28pt）：改进 YOLOv11 语义分割模块

左侧文字（14pt）：

**核心创新：C3k2_DFF 双域动态融合模块**
- 将原始 C3K2 模块重构为 C3k2_DFF_1 与 C3k2_DFF_2
- DFF（Dual-Domain Feature Fusion）采用并行双分支：通道注意力 + 空间注意力
- 1×1 卷积生成动态融合权重，而非固定比例

**协同模块：IEMA 注意力机制**
- 特征解耦：Identity 支路（边缘细节）+ C3K/Bottleneck 支路（语义特征）
- 特征增强：局部对比度增强 → 放大模糊边缘的灰度梯度
- 特征融合：跨尺度动态权重融合

**SCDown 轻量化组件**：在提升性能的同时控制模型参数

**图片**：`改进模型.png` 和 `语义分割模型.png`

**设计**：左右分栏——左边文字，右边放改进结构图。突出"C3k2_DFF"和"IEMA"两个核心模块名称。

## 第 10 页：GDX 临床数据模型

**类型**：技术深度页

**文字内容**：

大标题（28pt）：GDX 集成学习——改进的临床特征融合

**核心创新**（14pt）：

| 改进点 | 内容 |
|--------|------|
| 基模型互补 | XGBoost（稀疏特征 + 正则化） + GradientBoosting（连续特征 + 简单非线性） |
| 堆叠集成 | 元学习器（随机森林）自动学习基模型权重分配，替代简单平均 |
| 自动化调参 | GridSearchCV 系统搜索超参数空间 |
| 早停机制 | 验证集 AUC 连续 3 次不提升即终止训练，防止过拟合 |

**图片**：`临床实验数据图/各个特征的琴谱图.png`（特征分布）+ `论文中图 7 的流程图`（如果能提取）

**设计**：上方放特征分析图（琴谱图/热力图），下方用四列卡片列举四个改进点。技术突出"堆叠"和"自动化"两个关键词。

## 第 11 页：多维度融合策略

**类型**：过渡/逻辑页

**文字内容**：

大标题（28pt）：多维度融合——1+1+1 > 3

融合流程（14pt）：

```
维度一：超声图像语义特征（改进YOLOv11 → 病灶区域定位）
        +
维度二：病灶图像分类特征（EfficientNet → 良恶性判断）
        +
维度三：临床结构化数据（GDX → LV-GLS等指标分析）
        ↓
    Stacking 堆叠集成（基学习器 + 随机森林元分类器）
        ↓
    门控机制 → 过滤噪声数据 → 最终诊断
```

**设计**：三个维度用三个不同颜色的框从上到下排列，最后汇聚到一个中心节点"HTR-Net 融合决策"。视觉上强调"三个维度 → 一个决策"的设计逻辑。
