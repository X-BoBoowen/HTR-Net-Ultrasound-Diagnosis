# PPT 制作操作指南

## 一、制作前准备

1. 打开 PowerPoint，新建空白文档
2. 设置幻灯片大小：**设计 → 幻灯片大小 → 宽屏 16:9**
3. 配色方案（自定义主题色）：
   - 主色：深蓝 `#1A3A5C`（标题、强调）
   - 辅色：浅蓝 `#3A7BD5`（链接、图标）
   - 强调：橙红 `#E05030`（关键数字高亮）
   - 正文：深灰 `#333333`
   - 背景：纯白 `#FFFFFF`
   - 卡片背景：浅灰 `#F5F7FA`
4. 字体：全部使用 **微软雅黑**（中文）+ **Arial**（英文/数字）

## 二、逐页制作指南

### 第 1 页：封面

```
操作步骤：
1. 右键 → 设置背景格式 → 纯色填充 → 深蓝 #1A3A5C
2. 插入文本框，输入标题（36pt 白色加粗居中）
3. 插入副标题（20pt 浅蓝 #3A7BD5）
4. 底部插入个人信息（14pt 白色）
5. 可选：插入 论文图片/整体架构.png → 置于底层 → 透明度 80%
```

### 第 2 页：目录

```
操作步骤：
1. 左侧用形状画一个深蓝竖条（宽约 1cm）
2. 右侧用 7 行文本框列出目录项
3. 当前项深蓝色，其余灰色
4. 每项目前加数字圆圈（插入 → 形状 → 椭圆 + 文字）
```

### 第 3 页：临床痛点

```
操作步骤：
1. 插入 → 形状 → 矩形（圆角）→ 三个水平排列
2. 每个卡片内放：图标（插入 → 图标）、标题（16pt 加粗）、说明文字（12pt）
3. 卡片配色：浅蓝底、深蓝文字
4. 底部红色横条放总结语
```

### 第 4 页：整体架构

```
操作步骤：
1. 最重要的页，架构图占页面面积的 70%
2. 插入 论文图片/整体架构.png → 拉大居中
3. 底部用三个半透明矩形卡片解释三个阶段
4. 关键数字用橙红色加粗（85.7%, ~85%, 0.88）
```

### 第 5 页：分割模型改进

```
操作步骤：
1. 左右分栏：文字 40% + 图片 60%
2. 左侧：标题 + 正文（14pt）
3. 右侧：插入 改进模型.png 或 语义分割模型.png
4. 关键术语"C3k2_DFF""IEMA"用深蓝加粗
```

### 第 6 页：分割效果

```
操作步骤：
1. 顶部大标题
2. 三个关键数字用大号橙色字体横排在上方
3. 主体：插入 分割效果图展示.png，拉满页面宽度
4. 图片下方可加小字标注"原图 / Ground Truth / Ours"
```

### 第 7 页：消融实验（需自制图表）

```
操作步骤：
1. 打开 Excel，导入 制图数据-消融实验.csv
2. 选中 N1-N8 和 mIoU 两列 → 插入柱状图
3. 柱状图样式：N1-N7 浅蓝色，N8 橙红色
4. 在每个柱子上方标注 mIoU 数值
5. 截图或复制到 PPT，放在左侧
6. 右侧放一个表格展示 N8 的参数和计算量
7. 箭头标注"性能↑ 7.85pp 参数↓ 0.2M"
```

### 第 8-9 页：分类模型

```
操作步骤：
1. 左侧放分类模型设计思路（4 行文字）
2. 右侧放训练损失曲线（分类模型训练损失曲线.png）
3. 第 9 页放混淆矩阵 + 预测结果展示
```

### 第 10 页：GDX 模型

```
操作步骤：
1. 上方占页面 50%，放临床特征分析图（琴谱图 + 热力图并排）
2. 下方占页面 50%，放四列改进点卡片
3. 四个改进点用图标 + 标题 + 一句话说明
```

### 第 12 页：最终结果（最重要）

```
操作步骤：
1. 核心：表格。用 PPT 自带表格工具制作
2. 表格 3 行（GLS / TR-Net / HTR-Net）× 4 列（训练AUC / 验证AUC / 敏感性 / 特异性）
3. HTR-Net 行用深蓝色填充，数字白色加粗
4. 表格下方用箭头标注 HTR-Net vs GLS 的提升幅度
5. 右下角放 PR 曲线小图
```

### 第 14 页：总结

```
操作步骤：
1. 背景：深蓝 #1A3A5C，文字全白
2. 四大贡献纵向排列，每个前面有一个大数字（① ② ③ ④）
3. 数字用橙红 #E05030 醒目
4. 底部一行：IEEE BIBM 2026 在投 + GitHub 链接
```

### 第 15 页：致谢

```
操作步骤：
1. 白色背景，中间大号"谢谢！请各位老师批评指正"
2. 下方小字放联系方式 + GitHub
```

## 三、PPT 美化原则

1. **每页最多 3 种字号**：标题 28pt + 正文 14-16pt + 注释 10-12pt
2. **每页核心信息不超过 3 点**：如果发现写了 5 行字，拆成两页
3. **图片要出血（bleed）**：关键图拉到页面边缘，不留白边
4. **数字用大号橙色**：85.7%、0.88、7.85pp 这些数字用 24pt 以上橙红色
5. **动画极简**：只用"淡入"和"推入"两种，不要用旋转/弹跳
6. **对齐**：所有元素必须对齐到网格线（视图 → 网格线）

---

## 四、常犯错误提醒

| 错误 | 正确做法 |
|------|---------|
| 文字太多，评委看不清 | 每页控制在 50 字以内 |
| 图片模糊/太小 | 原图分辨率保证 1920×1080 以上 |
| 表格数据太多，找不到重点 | 核心数字加粗加色，非核心灰色 |
| 念稿逐字读 | PPT 是视觉辅助，讲稿是补充细节 |
| 技术术语不解释 | 首次出现的术语（如 C3k2_DFF）必须一句话解释 |
| 全部黑白色 | 关键信息一定要有颜色层级 |
